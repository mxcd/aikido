// llm/client.go

package llm

import (
    "context"
    "encoding/json"
)

// EventKind identifies the kind of a streaming event.
type EventKind int

const (
    EventTextDelta EventKind = iota
    EventToolCall  // emitted only after fragments fully assembled
    EventUsage
    EventError
    EventEnd       // always last; channel closes after
)

// Event is one streaming event emitted by a Client.
type Event struct {
    Kind  EventKind
    Text  string    // EventTextDelta
    Tool  *ToolCall // EventToolCall
    Usage *Usage    // EventUsage
    Err   error     // EventError
}

// Request is one provider call.
type Request struct {
    Model          string
    Messages       []Message
    Tools          []ToolDef
    MaxTokens      int
    Temperature    float32
    ThinkingEffort string // "", "low", "medium", "high" ŌĆö passthrough
    StopSequences  []string
}

// Client is the only interface a provider must implement.
type Client interface {
    Stream(ctx context.Context, req Request) (<-chan Event, error)
}
```

package llm

import "errors"

// ErrLLM is the package's sentinel base. Every typed error wraps it so callers
// can blanket-catch any llm-layer error with errors.Is(err, llm.ErrLLM).
var ErrLLM = errors.New("llm")

// Provider-classified errors. Each wraps ErrLLM. Providers wrap these with
// context: fmt.Errorf("openrouter: rate limit: %w", llm.ErrRateLimited).
var (
    ErrAuth           = fmt.Errorf("auth: %w", ErrLLM)
    ErrRateLimited    = fmt.Errorf("rate limited: %w", ErrLLM)
    ErrServerError    = fmt.Errorf("server error: %w", ErrLLM)
    ErrInvalidRequest = fmt.Errorf("invalid request: %w", ErrLLM)
    ErrUnknownModel   = fmt.Errorf("unknown model: %w", ErrLLM)
)
```

The shape is: one private sentinel (`ErrLLM`), every typed error wraps it via `fmt.Errorf("...: %w", ErrLLM)`. Callers can blanket-detect `errors.Is(err, llm.ErrLLM)` *or* test specific kinds via `errors.Is(err, llm.ErrRateLimited)`. Provider implementations (W2/W3) wrap these one more level: `fmt.Errorf("openrouter: %w", llm.ErrRateLimited)`. ### `Collect` reference implementation (~25 lines)

```go
// llm/helpers.go

func Collect(ctx context.Context, c Client, req Request) (text string, calls []ToolCall, usage *Usage, err error) {
    ch, err := c.Stream(ctx, req)
    if err != nil {
        return "", nil, nil, fmt.Errorf("collect: stream: %w", err)
    }
    var b strings.Builder
    for {
        select {
        case <-ctx.Done():
            return b.String(), calls, usage, ctx.Err()
        case ev, ok := <-ch:
            if !ok {
                return b.String(), calls, usage, nil
            }
            switch ev.Kind {
            case EventTextDelta:
                b.WriteString(ev.Text)
            case EventToolCall:
                if ev.Tool != nil {
                    calls = append(calls, *ev.Tool)
                }
            case EventUsage:
                usage = ev.Usage
            case EventError:
                return "", calls, usage, ev.Err
            case EventEnd:
                return b.String(), calls, usage, nil
            }
        }
    }
}
```

**Estimated LOC:** ~320 go + ~220 test | **Estimated effort:** 8 hours focused | **Status:** not started

**Goal.** Call OpenRouter end-to-end and produce a final completion. `Stream` is implemented as a single non-streaming HTTP call internally; the returned channel emits one `EventTextDelta` (entire content), one `EventUsage`, one `EventEnd`. ### W2 ŌåÆ W3 split (load-bearing)

The signature `func (c *Client) Stream(ctx context.Context, req llm.Request) (<-chan llm.Event, error)` does **not** change between W2 and W3. The implementation does:

| Aspect | W2 implementation | W3 implementation |
|---|---|---|
| Request body | `"stream": false` | `"stream": true` |

| Usage | One `EventUsage` from response.usage | One `EventUsage` from final-chunk usage |
| Retry | None in W2 (left as TODO in retry.go) | `retry.Do` wrapping stream-start in W3 per [OPENROUTER-DETAILS.md "Retry policy"](OPENROUTER-DETAILS.md) |
| Errors | Status-code classification at HTTP level | Same + mid-stream error envelope handling |

Why this split: W2 proves the wire shape (request body, auth header, response decode, error classification) end-to-end with the smallest possible surface, against fixtures that don't need an SSE parser. W3 swaps in the SSE machinery without re-litigating the type layer.

### `Stream` shape for W2

The W2 implementation is a synchronous POST that decodes one JSON response and replays it through a buffered channel. Mirrors the request-build pattern from [REFERENCE-CODE.md ┬¦5](REFERENCE-CODE.md) (cite `asolabs/hub/internal/client/openrouter/client.go` `doRequest` shape). Sequence:

1. Validate `req.Model != ""` (else `llm.ErrInvalidRequest`).
2. `body := json.Marshal(toAPIRequest(req, false /* stream */))`.
3. `http.NewRequestWithContext(ctx, POST, baseURL+"/chat/completions", body)` with headers `Content-Type`, `Authorization: Bearer`, optional `HTTP-Referer`, `X-Title`.
4. `httpClient.Do`. Non-200 ŌåÆ `classifyStatus(resp.StatusCode, body, retry-after)`. Return error directly; no channel.
5. Decode `chatResponse`. If `decoded.Error != nil`, return wrapped `llm.ErrServerError`.
6. Spawn goroutine: emit `EventTextDelta` for full content (string-unmarshal of `msg.Content`), `EventToolCall` per `msg.ToolCalls[i]`, `EventUsage` from `decoded.Usage` (via `fromAPIUsage`), `EventEnd`. Close channel.

`toAPIRequest` takes a `stream bool` parameter so W3 flips it to `true` without changing the signature. Retry wrapping (`retry.Do`) is left for W3.

### `assembleToolCalls` algorithm ŌĆö Go reference

```go
// llm/openrouter/sse.go

// merge applies one streamDelta's tool_calls fragments into the buffer.
// Idempotent semantics: empty fields don't overwrite non-empty ones.
func (b *toolCallBuffer) merge(frags []toolCallFragment) {
    for _, f := range frags {
        p, ok := b.partials[f.Index]
        if !ok {
            p = &partialToolCall{}
            b.partials[f.Index] = p
        }
        if f.ID != "" && p.id == "" {
            p.id = f.ID
        }
        if f.Function.Name != "" && p.name == "" {
            p.name = f.Function.Name
        }
        // Arguments always concatenate ŌĆö fragments are designed to append.
        if f.Function.Arguments != "" {
            p.args.WriteString(f.Function.Arguments)
        }
    }
}

// emitInIndexOrder walks the buffer and produces ToolCall events in ascending
// index order. Called once per choice when finish_reason arrives.
// Defensive: if a partial has no id (rare malformed provider), synthesize
// "call_<index>" so downstream tool-result messages can reply correctly.
func (b *toolCallBuffer) emitInIndexOrder() []llm.ToolCall {
    if len(b.partials) == 0 {
        return nil
    }
    indexes := make([]int, 0, len(b.partials))
    for i := range b.partials {
        indexes = append(indexes, i)
    }
    sort.Ints(indexes)

    out := make([]llm.ToolCall, 0, len(indexes))
    for _, i := range indexes {
        p := b.partials[i]
        id := p.id
        if id == "" {
            id = fmt.Sprintf("call_%d", i)
        }
        out = append(out, llm.ToolCall{
            ID:        id,
            Name:      p.name,
            Arguments: p.args.String(),
        })
    }
    b.partials = make(map[int]*partialToolCall) // reset for next turn (defensive)
    return out
}
```

The emit-at-finish-reason rule: `streamSSE` calls `emitInIndexOrder` exactly once per choice, when the choice's `finish_reason` (regardless of value: `"tool_calls"`, `"stop"`, `"length"`, `"error"`) arrives. This handles the edge case in [OPENROUTER-DETAILS.md "Edge cases"](OPENROUTER-DETAILS.md) where `finish_reason: "stop"` arrives with non-empty buffer.

### `streamSSE` pipeline (algorithm sketch)

`streamSSE` is the goroutine body for the W3 `Stream` method. It owns the `out` channel and closes it on exit. Pipeline:

1. Construct `parser := sseparse.NewParser(body)` and `buf := newToolCallBuffer()`. Track `pendingUsage *llm.Usage` (nil sentinel).
2. Loop: select on `ctx.Done` (emit `EventError(ctx.Err) + EventEnd`, return). Otherwise call `parser.Next()`.
3. On `io.EOF` (clean stream end without `[DONE]`): drain `buf.emitInIndexOrder()` as `EventToolCall`s, then `pendingUsage` as `EventUsage`, then `EventEnd`.
4. On other parse error: `EventError(err) + EventEnd`.
5. On terminator (`[DONE]`): same drain as EOF case.
6. On nil line (comment / blank): continue.
7. On data line: `json.Unmarshal(line, &chunk)`. Unknown fields ignored (permissive). Marshal failure ŌåÆ continue (defensive).
8. If `chunk.Error != nil`: emit `EventError` wrapping `llm.ErrServerError` with `chunk.Error.Message` (per [OPENROUTER-DETAILS.md "Mid-stream errors"](OPENROUTER-DETAILS.md)), then `EventEnd`. Do NOT retry.
9. If `chunk.Usage != nil`: store in `pendingUsage` (do not emit yet ŌĆö usage chunk may precede final chunk on some providers).
10. For each `chunk.Choices[i]`:
    - `Delta.Content != ""` ŌåÆ emit `EventTextDelta`.
    - `Delta.ToolCalls` non-empty ŌåÆ `buf.merge(...)`.
    - `FinishReason != nil` ŌåÆ drain `buf.emitInIndexOrder()` as `EventToolCall`s. Do NOT emit `EventEnd` yet ŌĆö usage may still arrive on a trailing chunk.
11. Every channel send is wrapped in `select { case out <- ev: case <-ctx.Done(): return }` to prevent producer-leak on slow consumers.

Order invariant per [OPENROUTER-DETAILS.md "Where usage lives"](OPENROUTER-DETAILS.md): tool calls emit at finish_reason; usage may arrive same-chunk or trailing; both emit before `EventEnd`.

### `internal/sseparse/parse.go` ŌĆö shared parser

OpenRouter-agnostic line parser; future providers (`llm/anthropic` in v2) reuse it directly. Public surface:

```go
package sseparse

type Parser struct{ /* ... */ }

func NewParser(r io.Reader) *Parser
// Next reads one logical event payload.
//   - (data, false, nil)    : JSON-encoded chunk in `data`
//   - (nil,  true,  nil)    : the [DONE] terminator
//   - (nil,  false, io.EOF) : clean end of stream without [DONE]
//   - (nil,  false, err)    : parse / read error
// Comment lines (prefix `:`) and blank separators are absorbed silently.
func (p *Parser) Next() (data []byte, terminator bool, err error)
```

Implementation: `bufio.Scanner` with 1 MiB buffer (default 64 KiB is too small for large tool-arg JSON). Strips `\r` and `\n` automatically per `bufio.ScanLines`. Lines starting with `:` are SSE comments (e.g. `: OPENROUTER PROCESSING`) and skipped. Lines starting with `data: ` are stripped of prefix and returned. Lines that don't match either are skipped (permissive policy per [PLAN.md](../PLAN.md) W2 risks). Constants: `dataPrefix = "data: "`, `doneMarker = "[DONE]"`.

### `retry.go` ŌĆö stream-start guard

`streamStart(ctx, httpReq) (*http.Response, error)` wraps `c.httpClient.Do` inside `retry.Do(ctx, *c.retry, fn)`. The closure:

1. Rewinds the request body via `httpReq.GetBody()` (set when caller built `httpReq` with `bytes.NewReader`).
2. Calls `c.httpClient.Do(httpReq)`.
3. On 429: drain + close body; return classified `llm.ErrRateLimited` (with `Retry-After` parsed). Retriable.
4. On 5xx (excluding 501/505 which are client-fault permanent): drain + close body; return classified `llm.ErrServerError`. Retriable.
5. On other non-200: read body for diagnostic; return classified non-retriable error.
6. On 200: capture `r` into outer `resp`; return nil.

`defaultRetryPolicy()` returns `retry.DefaultPolicy()` with `RetryOn = errors.Is(err, llm.ErrRateLimited) || errors.Is(err, llm.ErrServerError)`. If `Retry-After` is set on a 429 response, the wrapper sleeps for that duration instead of computing the next backoff (per [OPENROUTER-DETAILS.md "Retry policy"](OPENROUTER-DETAILS.md)).

**Stream-start semantics.** The retry guard is **structural**, not a boolean: `retry.Do` only wraps `streamStart`, not `streamSSE`. Once `streamStart` returns a non-nil 200 `*http.Response` and the body is handed to `streamSSE`, retries are physically impossible ŌĆö the body is already being read by the SSE parser. The "firstByteReceived" semantic is the goroutine boundary itself; there is no flag to flip. Tests prove this via the `mid_stream_error.sse` fixture: status is 200, error envelope arrives mid-stream, retry must NOT happen (only 1 HTTP request observed).

### Risks (concrete mitigations)

- Risk: SSE parser breaks on unknown line shapes. Mitigation: permissive parser (skip unknown lines); fixtures explicitly test malformed input.
- Risk: Tool-call assembly emits incomplete JSON if the model truncates mid-fragment. Mitigation: `emitInIndexOrder` does not validate JSON (per [OPENROUTER-DETAILS.md "Tool-call fragmentation" step 4](OPENROUTER-DETAILS.md)) ŌĆö `tools.Registry.Dispatch` validates at parse time; bad JSON surfaces as a tool error returned to the model, who can correct course (per ARCHITECTURE.md "Tool errors do not abort the loop").
- Risk: Retry-After parsing bug ŌĆö RFC 7231 allows integer seconds OR HTTP-date. Mitigation: `parseRetryAfter` tries integer first, falls back to HTTP-date (`http.ParseTime`), defaults to backoff if neither.
- Risk: Mid-stream goroutine leak if `out` channel is full and producer blocks forever. Mitigation: every producer write is wrapped in a `select { case out <- ev: case <-ctx.Done(): return }` block. Tested in TestStream_ContextCancellation.
- Risk: `httptest.Server` flushes whole response body before client reads. Mitigation: tests use `http.Flusher` after each chunk; the test harness's helper `serveSSE(t, fixturePath)` flushes per line to simulate real streaming.
- Risk: The 1 MiB scanner buffer is exceeded by a single tool-call fragment. Mitigation: extremely unlikely for OpenAI-compatible providers (max output 8K tokens ├Ś ~4 bytes/token Ōēł 32 KiB); if it ever surfaces, raise the cap. Add a TODO.
- Risk: Tool-call fragment without `name` ever ŌĆö the model emits `id` only. Mitigation: `emitInIndexOrder` synthesizes name as empty string; `tools.Registry.Dispatch` returns `ErrUnknownTool` for empty name; loop continues.

- `NewRegistry()` initializes the empty maps and a zero-length `order` slice.
- `Register` validates non-empty `def.Name` and non-nil `h`, takes the write lock, errors with `ErrDuplicateTool` on collision, inserts into `defs`/`handlers`, appends to `order`.
- `Defs()` takes the read lock and returns a fresh `[]llm.ToolDef` walking `order` (registration-order preserving).
- `Dispatch` takes the read lock to look up the handler; on miss returns wrapped `ErrUnknownTool`; on hit calls `h(ctx, json.RawMessage(call.Arguments), env)`. Lock is released before invoking the handler so handlers can re-enter the registry safely.
- `Has(name)` is a read-lock map probe.

Mirrors the `sync.RWMutex`-protected map pattern from `mxcd/go-basicauth/storage_memory.go` per [REFERENCE-CODE.md ┬¦11](REFERENCE-CODE.md).

### Schema-helper behavior

- `Object(props, required...)` builds `{type: "object", properties: props}`, adds `required: [...]` if non-empty, marshals to `json.RawMessage`. Marshal errors fall back to `{"type":"object"}` (defensive ŌĆö `map[string]any` with primitives can't error in practice).
- `String/Integer/Number/Boolean(description)` return `map[string]any{"type": <typename>, "description": description}`.
- `Enum(description, values...)` returns the String shape plus `"enum": values`.
- `Array(items, description)` returns `{type: "array", items: items, description: description}`.
- `SchemaFromType[T]()` instantiates a `jsonschema.Reflector{DoNotReference:true, AllowAdditionalProperties:false, RequiredFromJSONSchemaTags:true}`, calls `ReflectFromType(reflect.TypeOf(zeroT))`, marshals to `json.RawMessage`. Inlines all definitions (no `$ref`) for maximum provider compatibility.

### Risks (concrete mitigations)

- Risk: `invopop/jsonschema` produces non-OpenAI-compatible schemas for some Go types (`time.Time` with custom format). Mitigation: document that `SchemaFromType` is for "flat structs with primitive fields" (per ADR-005); complex types use the explicit form. Add a TODO to revisit if user demand surfaces.
- Risk: Registry is a `*Registry` (pointer) on `agent.Options.Tools`; callers might mutate it after `NewAgent`. Mitigation: per [REFERENCE-CODE.md Q9](REFERENCE-CODE.md), document that mutating between `Run`s is OK; mutating *during* a `Run` is undefined. The mutex protects against data race, but per-run consistency requires the caller's discipline.
- Risk: Tool name collisions between built-in tools (W7: `read_file`, `write_file`, ...) and caller's domain tools. Mitigation: `Register` returns `ErrDuplicateTool`; built-in registration via `RegisterVFSTools` happens after caller registration, surface this in the godoc as "register your tools first; built-in tools error out if you've shadowed them."
- Risk: Reflection helper carries an extra dep. Mitigation: per ADR-005, the dep is only pulled when `SchemaFromType` is called ŌĆö Go's per-package tracking ensures library users importing only `tools` (using explicit helpers) do not pull `invopop/jsonschema` transitively. Verified by `go mod why` after W4.

## Cross-wave invariants

These hold throughout W0-W4 (and remain in W5-W8). Violation should fail review.

- **No global state.** Every package's exported state lives on a struct constructed via `NewX(opts *Options)`. No `init()`, no package-level `var foo = ...` except sentinel errors.
- **Every constructor returns `(*X, error)`.** Validation errors are user-readable; no `%w`-wrap of stdlib errors at constructor time.
- **Every public type has a one-line godoc.** "What it is" first; "when to use" second; full prose only when non-obvious.
- **Compile-time interface conformance** as `var _ I = (*T)(nil)` at the top of the implementing file (per [REFERENCE-CODE.md aggregate insights](REFERENCE-CODE.md)).
- **JSON tags use snake_case** on wire types; public Go types use PascalCase fields.
- **Errors compose via `errors.Is`**; never `==` for sentinel detection.
- **Tests are race-clean.** `just test` runs `go test -race`; CI fails on race detector reports.
- **Dates in docs use `dd.MM.yyyy`.**
